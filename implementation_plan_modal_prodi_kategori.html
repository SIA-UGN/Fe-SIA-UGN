3. API Endpoint Mapping

ModalEndpointBody / Response
Program Studi Tidak ada endpoint khusus. Data id_program sudah ada dari login/user context. Bisa fetch dari endpoint existing jika tersedia. Tampilkan daftar program, user pilih untuk di-set di form topik
Kategori TA (Create) POST /api/lecturer/thesis/categories { name, description? } → 201 Created
Kategori (Read) GET /api/lecturer/thesis/categories Fetch list saat modal dibuka, refresh setelah simpan

4. Spesifikasi Komponen Modal

TambahProgramStudiModal

Props: isOpen, onClose, onSuccess

Fields:

• Nama Program Studi (input text, required)

• Kode Prodi (input text, optional)

• Jenjang (select: D3/S1/S2, optional)

State: formData, loading, error

Actions: Batal → onClose, Simpan → onSuccess + onClose

Trigger: Tombol icon "+" di pojok kanan header container

TambahKategoriTAModal

Props: isOpen, onClose, onSuccess

Fields:

• Nama Kategori (input text, required)

• Deskripsi (textarea, optional)

State: formData, loading, errors, apiError

API: POST /api/lecturer/thesis/categories

On success: refresh list kategori di parent, tutup modal

7. Validasi & Error Handling

Client-side

• Nama tidak boleh kosong (required)

• Max 255 karakter untuk name

• Tampilkan error di bawah input field (text merah, 12px)

Server-side (API 422)

• Parse errors object dari response

• Map ke field terkait

• Tampilkan toast/snackbar untuk error umum 500
